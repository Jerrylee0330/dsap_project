# [專題名稱]

## Proposal Report

### 動機與目標
在學習微積分的過程中，我常常會使用 Gemini、 Wolfram Alpha 或 Photomath 等強大的AI、網站、軟體等進行運算。然而，這些工具高度依賴網路連線，在台大這個網路不太好的大學的各種地方，你都有可能連不上這些網站或使用這些app，而我在使用這些工具幫助我解決問題時，也會遇到他們答不出來、回答錯誤或是GUI介面不直觀的問題。因此我想在我的電腦中做一個可以計算微積分的計算機，讓我可以在沒有網路的情況下，也能夠準確地算出我想要的答案。

本專案的目標是開發一款具備圖形化介面（GUI）、完全離線運作的微積分計算機。透過實作堆疊解析與抽象語法樹（AST），程式將能解讀人類輸入的數學算式，並運用「符號運算」進行精確求導。同時，針對數學上缺乏解析解（Closed-form solution）的積分難題，本專案將引入「數值逼近」作為退場機制，確保程式的強健性與實用價值。

### 預期功能
圖形化互動介面 (GUI)：打造直覺的操作面板，包含數字、變數 $x$、基礎運算子與微積分功能鍵，並具備算式狀態顯示與基礎防呆機制（如括號配對檢查）。

算式解析與基礎四則運算：能處理包含括號、加減乘除、次方及基本函數的中序表達式，並正確計算優先級。

符號微分引擎：使用者輸入方程式後，系統能透過內部語法樹的規則替換，精確推導出一階導函數的數學式。

定積分數值運算：針對積分問題，提供數值運算模式。使用者輸入積分上下界後，系統將透過多次走訪求值結合數值分析演算法，計算出精確的定積分面積。

### 使用技術
C++。

### 時程規劃
第 1 週：GUI 介面建置與事件處理使用 C++ GUI 框架（如 Qt 或 ImGui）完成計算機外殼排版。實裝所有功能按鍵，並確保點擊按鈕能將字元正確輸入至緩衝字串中。完成「清除 (AC)」與「退格 (Backspace)」邏輯。

第 2 週：四則運算解析 (Stack 應用)實作 Shunting-yard 演算法。利用 Stack (堆疊) 將中序表達式轉為後序表達式。實作後序表達式的數值求值，完成基礎計算機功能。

第 3-4 週：抽象語法樹 (AST) 與微分邏輯建構 AST (抽象語法樹)。將後序表達式轉化為樹狀結構節點。實作遞迴微分函數。針對 AST 的不同節點（Constant, Variable, Operator）套用微分規則。

第 5-6 週：符號積分與規則匹配實作基本的反導數規則（Antiderivative rules）。建立模式匹配邏輯，判斷 AST 結構是否符合特定的積分公式（例如多項式積分）。處理基本的符號積分輸出。

第 7-8 週：數值積分 Fallback 與系統優化針對無法以符號積分技巧處理的函數，開發「數值積分模式」。引導使用者輸入上下界，利用 Simpson's Rule (辛普森法) 或梯形法，透過不斷走訪 AST 求值來計算定積分。最終系統測試、邊界錯誤處理（除以零、括號不匹配）與專題文件撰寫。

### 與課程的關聯
堆疊 (Stack) 與佇列 (Queue) 的實務應用：解析使用者輸入的數學算式（中序表達式）是本專題的第一道關卡。專題中將實作調度場演算法 (Shunting-yard Algorithm)，利用 Stack 來嚴格控制運算子的優先權（先乘除後加減）與括號的匹配配對，並將解析結果轉換為後序表達式 (Postfix Expression)。這是線性資料結構最經典且核心的應用場景。

樹狀結構 (Tree) 的建構與操作：微積分運算的核心，在於將一維的字串算式，轉化為具備階層關係的抽象語法樹 (Abstract Syntax Tree, AST)。在此樹狀結構中，內部節點 (Internal Nodes) 代表運算子，葉節點 (Leaf Nodes) 代表變數與常數。這完美展現了 Tree 結構在處理非線性、具備從屬關係資料時的強大優勢。

遞迴 (Recursion) 與樹的走訪 (Tree Traversal)：本專題中的微積分核心引擎高度仰賴遞迴演算法。

數值求值： 透過對 AST 進行後序走訪 (Post-order Traversal)，由樹的底部向上遞迴計算，得出最終數值。

符號微分： 將微積分的連鎖律與乘法律轉化為程式邏輯，透過遞迴呼叫走訪各個子樹，針對不同節點型態進行複製、替換與重組，最終生成一棵全新的導函數 AST。這深刻體現了遞迴與樹狀結構的協同作用。

物件導向程式設計 (OOP) 與指標管理：在建構 AST 時，將運用 C++ 的物件導向特性（繼承與多型），設計基礎的 Node 類別並衍生出不同的子類別（如 OperatorNode, VariableNode）。同時，在微分過程與樹的重建中，動態配置與釋放節點將實質考驗對指標 (Pointer) 與記憶體管理的掌握度。
